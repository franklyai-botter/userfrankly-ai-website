# GitHub Issues / Arbeitspakete

## Issue 1 – Projektstruktur prüfen und Setup finalisieren

**Ziel:** Bestehendes Repository analysieren und Tech Stack bestätigen.

**Tasks:**

- Vorhandene Projektstruktur prüfen
- Package Manager identifizieren
- Tailwind/shadcn prüfen
- Routing-Struktur festlegen
- Content-Ablage planen

**Akzeptanzkriterien:**

- Projekt startet lokal
- Struktur ist dokumentiert
- Offene technische Risiken sind benannt

---

## Issue 2 – Content zentralisieren

**Ziel:** Inhalte aus dem Handoff-Paket in eine zentrale Content-Datei übertragen.

**Tasks:**

- `src/data/content.ts` oder `content.json` anlegen
- Navigation auslagern
- Services, Use Cases, Methode und SEO strukturieren

**Akzeptanzkriterien:**

- Keine Lorem-Ipsum-Texte
- Inhalte sind wiederverwendbar
- Navigation ist zentral pflegbar

---

## Issue 3 – Layout, Header und Footer bauen

**Ziel:** Grundlayout der Website erstellen.

**Tasks:**

- Footer

**Akzeptanzkriterien:**

- Footer enthält Kontakt- und Rechtelinks

## Issue 4 – Startseite umsetzen

## Issue 11 – SEO und Accessibility

**Ziel:** Basis-SEO und Zugänglichkeit verbessern.

**Tasks:**

- Meta Titles
- Meta Descriptions
- Semantische HTML-Struktur
- Alt-Texte
- Button Labels
- Keyboard Navigation prüfen

**Akzeptanzkriterien:**

- Jede Seite hat eigene Metadaten
- Keine offensichtlichen Accessibility-Fehler
- Gute Lighthouse-Basis
