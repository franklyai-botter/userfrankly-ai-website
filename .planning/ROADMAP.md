## Phase Details

### Phase 1: Hero Visual
**Goal**: Besucher sehen beim ersten Seitenaufruf ein visuell starkes Neural Network im Hero
**Depends on**: Nothing (visual polish on existing structure)
**Requirements**: HERO-01
**Success Criteria** (what must be TRUE):
  1. Das Neural Network ist auf dem Desktop-Hero klar auf der rechten Seite sichtbar — kein vollständiges Überdecken durch Gradient
  2. Der Gradient-Overlay lässt das Netzwerk durch, ohne den Lesetext zu beeinträchtigen
  3. Der visuelle Eindruck entspricht dem Design-Handoff
**Plans**: TBD
**UI hint**: yes

### Phase 2: Animationen
**Goal**: Elemente auf der gesamten Website verhalten sich lebendig beim Scrollen und bei Interaktion
**Depends on**: Phase 1
**Requirements**: ANIM-01, ANIM-02
**Success Criteria** (what must be TRUE):
  1. Sektionen und Karten auf allen Seiten faden sichtbar ein, wenn sie in den Viewport gescrollt werden
  2. Karten, Buttons und Links zeigen beim Hover eine subtile Reaktion (Skalierung, Glow oder Farbwechsel)
  3. Animationen laufen flüssig ohne Layout-Shifts oder Performance-Einbrüche
**Plans**: TBD
**UI hint**: yes

### Phase 3: Mobile
**Goal**: Die gesamte Website ist auf Smartphone vollständig bedienbar und visuell ansprechend
**Depends on**: Phase 2
**Requirements**: MOB-01, MOB-02, MOB-03, MOB-04, MOB-05
**Success Criteria** (what must be TRUE):
  1. Navigation auf Smartphone öffnet sich als Hamburger-Menü oder Drawer und alle Links sind tippbar
  2. Hero-Section auf Smartphone zeigt Schrift und Neural Network in einem angepassten, lesbaren Layout
  3. Alle Grid-Layouts (3-, 4-, 5-spaltig) brechen auf Mobile korrekt in 1 oder 2 Spalten zusammen
  4. Footer-Links auf Smartphone haben ausreichende Touch-Targets (mindestens 44px)
  5. Unterseiten-Header mit Neural Network Background sehen auf Smartphone gut aus — kein Clipping oder Verzerrung
